How view transitions work

    
    
      
  • One callback, two snapshots. document.startViewTransition(updateDOM) screenshots the page, runs updateDOM, screenshots again, and animates between snapshots.
    • 
      
  • Names match elements across states. Set view-transition-name: hero on the source and on the destination element. The browser ties them into a single pair (called a group) and animates position + size automatically.
    • 
      
  • Customize per name. Target ::view-transition-old(name) and ::view-transition-new(name) to override the default cross-fade — slide, flip, zoom, anything CSS animations can do.
    • 
      
  • Names must be unique at any moment. Two elements sharing a name in the same state break the transition. Use contain: layout if the element jumps around layout.
    • 
      
  • Respect prefers-reduced-motion. Wrap your custom keyframes in a motion-safe media query so vestibular users get the default fade (or no animation) instead.
