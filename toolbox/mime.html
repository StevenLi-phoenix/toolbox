When charset matters

              
Use a charset for textual content that browsers or APIs should decode as text.

            

            

              
Good fits

              
text/plain, text/html, text/css, application/json, application/javascript, image/svg+xml, application/xml.

            

            

              
Usually skip it

              
Binary assets like image/png, application/pdf, application/zip, font/woff2, video/mp4.

            

          

        


        

          

            

              
Security reminders

            

            

              
XSS risk Serving user-controlled SVG or HTML inline can execute active content. Consider download-only delivery or sanitization.

              
Sniffing Add X-Content-Type-Options: nosniff for scripts and styles so browsers do not guess.

              
Uploads Store both original filename and trusted server-detected MIME type; never trust extension alone.
