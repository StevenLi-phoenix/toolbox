Bash Brace Expansion

Type a bash brace pattern and see exactly what the shell would expand it to — without running anything. Supports comma lists {a,b,c}, numeric ranges {1..10}, character ranges {a..z}, ranges with steps {0..20..2}, zero padding {001..010}, and arbitrary nesting {a,b{1..3},c}.

Notes

  • This implements the GNU bash brace-expansion grammar — it does not perform globbing, parameter expansion, or word splitting.
  • Adjacent groups produce a Cartesian product: {a,b}{1,2} → a1 a2 b1 b2.
  • Numeric ranges respect zero-padding when either endpoint is padded: {01..05} → 01 02 03 04 05.
  • Character ranges work on single ASCII letters: {a..f} → a b c d e f. They go in either direction.
  • An optional step can reverse direction: {10..1..2} → 10 8 6 4 2.
  • If a brace group can't be parsed (single element, no .., no comma), it is left intact like bash does.
  • To prevent runaway expansion, output is capped at 100,000 entries.
