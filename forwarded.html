When to trust these headers

  • Only trust forwarding headers from proxies or CDNs you actually control.
  • Rightmost entries usually represent the nearest upstream proxies; count trust from the right, not the left.
  • If untrusted clients can connect directly to the app, they can spoof X-Forwarded-For and Forwarded.
  • Forwarded is standardized, but many stacks still rely on legacy headers. Handle both when debugging.
  • Prefer explicit allowlists or exact hop counts over trust proxy = true in production.
