ARN Format

arn:partition:service:region:account-id:resource-id

arn:partition:service:region:account-id:resource-type/resource-id

arn:partition:service:region:account-id:resource-type:resource-id

Segment Rules

  • partition — aws, aws-cn (China), aws-us-gov (GovCloud).
  • service — AWS service namespace (e.g. s3, iam, lambda, ec2).
  • region — AWS region like us-east-1. Omitted (empty) for global services like IAM and S3 bucket ARNs.
  • account-id — 12-digit AWS account number. Omitted for some resources (e.g. public S3).
  • resource-id — The resource identifier. May include type and ID, separated by / or :, depending on the service.

Wildcards in IAM Policies

You can use * (zero or more characters) and ? (one character) in ARN fields in IAM policies. For example, arn:aws:s3:::my-bucket/* matches every object in my-bucket.
