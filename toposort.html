About topological sort

  

    A topological sort of a directed graph is an ordering of nodes where every edge u → v places u before v. It only exists if the graph is a DAG (directed acyclic graph) — a cycle makes the order undefined.
    Kahn's algorithm (1962) is the iterative version: collect every node with in-degree 0, emit them, remove their outgoing edges, repeat. The depth-first variant produces the order by post-order traversal in reverse. Both run in O(V + E).
  

  

    Where you'll meet it: build systems (Make, Bazel, Nix), package managers (npm, Cargo), task schedulers (Airflow, Dagster, Tekton), course prerequisites, spreadsheet recalc, linker symbol resolution, and data-flow IRs in compilers. If a build complains about a "circular dependency," it ran a topological sort and gave up at the surviving nodes — the same ones this tool will highlight.
