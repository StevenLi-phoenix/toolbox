CSS light-dark() Playground

The CSS light-dark(LIGHT, DARK) function returns the first value under a light color-scheme and the second under a dark one — letting you define one set of custom properties that adapts automatically, with no @media (prefers-color-scheme) blocks. Tips

    
    
      
  • Always set color-scheme on :root — without it, light-dark() resolves to the light value. color-scheme: light dark lets the UA follow the user's preference.
    • 
      
  • Forced manual themes: set color-scheme: light on a .theme-light class and color-scheme: dark on .theme-dark — the same light-dark() tokens automatically swap.
    • 
      
  • Don't nest light-dark() inside light-dark() — only the outermost call resolves cleanly. Keep nesting in custom properties, not in the function itself.
    • 
      
  • Form controls (checkboxes, scrollbars, inputs) inherit color-scheme automatically — so explicitly setting it also fixes browser-rendered UI without any color overrides.
    • 
      
  • Fallback chain: background: var(--bg-fallback); background: light-dark(white, #111); — older browsers use the first, modern ones override with the second.
