Stripe Test Card Numbers

Reference of Stripe's documented test card numbers — for use only against Stripe's test mode (publishable key starting pk_test_). Use any future expiry date and any 3-digit CVC (4 digits for Amex). Click any card number to copy. Real cards declined here will be rejected by Stripe.

Testing notes

  • Use any future expiration date (MM/YY) — e.g. 12/34.
  • Use any 3-digit CVC (American Express: 4 digits).
  • Use any postal / ZIP code (42424 works); AVS-failure cards override that.
  • 3DS test cards may require completing the on-page challenge popup. Use the “Complete authentication” button to succeed and “Fail authentication” to decline.
  • For declines, the charge.failure_code on the resulting charge object matches the scenario column.
  • To test a dispute lifecycle, charge 4000 0000 0000 0259 then go to the Dashboard and submit evidence — the dispute will be auto-won or auto-lost based on the card.
  • Wallets / SEPA / iDEAL / BACS test tokens are not raw card numbers and are not listed here; use Stripe's published pm_card_* PaymentMethod IDs.

Last updated against Stripe documentation as of 2025. Always cross-check at stripe.com/docs/testing if a card changes behaviour.
