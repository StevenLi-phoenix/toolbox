CSS field-sizing Generator

The CSS field-sizing property lets form controls — <textarea>, <input>, <select> — size themselves to their content instead of their UA-defined default box. Why this matters

    
    
      
  • Replaces the scrollHeight hack. The classic JS pattern (el.style.height = 'auto'; el.style.height = el.scrollHeight + 'px') thrashes layout and breaks on paste. field-sizing: content does it intrinsically.
    • 
      
  • Sizes <select> too. Selects with long options finally stop overflowing their container; the dropdown shrinks back when a shorter option is picked.
    • 
      
  • Always set min-* and max-*. An empty textarea with field-sizing: content can collapse to zero height — min-height: 2lh keeps it usable.
    • 
      
  • ch and lh units pair perfectly. They scale with the form control's font, so a 40-character textarea stays ~40 characters wide whether the font is 14px or 24px.
    • 
      
  • Works with resize. Users can still drag the corner to override; field-sizing only sets the intrinsic preferred size.
